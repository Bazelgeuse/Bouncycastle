ssh 방식은 Github 에서 사용자 특정 폴더의 키값을 검사하여 
해당 사용자가 인증된 사용자인지 체킹하는 보안 방식이다.

즉 ,Git 통신을 위한 자동화를 위해서는 ssh 통신 방식은 반드시 필요하다.

먼저, gitbash 프로그램을 다운받아서 설치 한다.
설치후 Git Bash 명령 프롬포트 창을 연후 아래 명령어 입력한다.
공개키 만들기
먼저 본인의  홈디렉토리에서 .ssh디렉터리에 들어가서 공개키가 존재하는지 확인한다.
만약 키가 존재할경우 id_rsa , id_rsa.pub와 같은 파일이 있을것이다.
하지만 공개키가 없을경우 파일은 커녕 .ssh디렉터리조차 없을 수 있다.
따라서 공개키를 먼저 만든다.
ssh-keygen명령어를 이용해서 공개키를 만들 때 깃허브에 가입한 이메일을 적어준다
그 다음 엔터를 두번눌러서 아래의 패스프레이즈를 스킵해주도록한다.
다음에 .ssh디렉터리로 가면 .pub파일이 생긴 것을 확인할 수 있다.
앞서 만든 공개키를 cat명령어를 사용하여 .pub파일의 내용을 출력한 다음 이를 복사하도록 한다.
복사한 키를 이용하여 밑에서 사용한다.
먼저 https://github.com 접속한다.
 - 로그인한다.
 - 우측상단 사용자 클릭하여 Settings 클릭한다.
 - SSH and GPG keys 메뉴 클릭한다.
 - New SSh key 버튼 클릭하여 아까 C:\Users\사용자\.ssh 하위 폴더에 생성된 id_rsa.pub 파일의 내용을 복사하여 등록한다.
마지막으로 Git bash 열어서 소스 있는 폴더까지 이동 후 초기화 작업후 push 한다
